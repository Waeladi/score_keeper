# Score Keeper - Claude Code Project Guide

## Project Overview
A Flutter card game score-tracking app for Android (phones and tablets). Published on Google Play Store.
- **Package**: `com.waelapps.score_keeper`
- **Display Name**: "Game Scores"
- **Platform**: Android only (phone + tablet)
- **Min SDK**: Flutter 3.7.0, Dart 3.7.0, Java 17

## Tech Stack
- **Framework**: Flutter with Material 3
- **State Persistence**: SharedPreferences (JSON serialization)
- **Analytics**: Firebase Analytics, Firebase Crashlytics
- **Charting**: fl_chart
- **Build**: Gradle Kotlin DSL, GitHub Actions CI/CD

## Project Structure
```
lib/
├── main.dart              # Entry point: Firebase init + runApp + re-exports
├── app.dart               # ScoreKeeperApp MaterialApp widget
├── firebase_options.dart  # Auto-generated Firebase config (do not edit)
├── screens/
│   ├── main_screen.dart   # MainScreen: bottom nav, state orchestration
│   ├── scoring_screen.dart # Score entry with portrait/landscape layouts
│   ├── history_screen.dart # Score history DataTable with revert
│   ├── new_game_screen.dart # New game / reset game
│   └── graph_screen.dart  # Score progression chart + legend
├── widgets/
│   ├── player_score_input.dart # Unified player score input (portrait + landscape)
│   └── score_chart.dart   # Line chart widget using fl_chart
├── models/
│   └── game_state.dart    # Game state model with JSON persistence
└── utils/
    └── constants.dart     # Colors, text styles, layout constants, maxPlayers
```

## Architecture Notes
- State management: setState with callbacks, orchestrated in MainScreen
- GameState model handles serialization and persistence to SharedPreferences
- Orientation handling: Native Android (MainActivity.kt) locks phones to portrait, tablets can rotate
- All screens support portrait/landscape layouts via `MediaQuery.of(context).orientation`
- Bottom navigation with 4 tabs: Home (Scoring), History, New Game, Graph
- `main.dart` re-exports key classes for backward-compatible test imports

## Important Conventions
- Player count: 2-4 players per game, max defined in `AppConstants.maxPlayers`
- Arrays are sized to `maxPlayers` (4) regardless of actual player count
- Score sign is toggled via +/- button (not typed as negative)
- History revert uses long-press gesture on DataTable rows, via `onRevertToRound` callback
- Firebase Analytics logs navigation events as 'nav_click'

## Files NOT to Edit
- `firebase_options.dart` — auto-generated by FlutterFire CLI
- `android/app/google-services.json` — Firebase config
- `.github/workflows/` — CI/CD pipelines (edit with care)

## Testing
- Unit tests: `test/unit_tests/game_state_test.dart`
- Widget tests: `test/widget_tests/scoring_screen_test.dart`, `test/widget_test.dart`
- Integration tests: `integration_test/app_test.dart`, `integration_test/trix_test.dart`
- Diagnostic utility: `test/diagnostic_test.dart`
- Widget tests that need portrait mode must set `tester.view.physicalSize` (default is 800x600 = landscape)
